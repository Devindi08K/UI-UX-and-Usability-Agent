UI/UX Usability Agent - Setup Instructions
What this project is
This is a research project for an undergraduate dissertation at SLIIT.
The component being built is the UI/UX & Usability Agent — one part
of a larger Multi-Agent LLM-Based Framework for Early-Phase Software
Engineering of Novice Developers.
What this agent does (step by step)

Takes a JSON file of software requirements as input
Sends those requirements to a local LLM (Ollama + Llama3)
The LLM generates an HTML + Tailwind CSS UI prototype
A custom evaluation engine scores that HTML on 3 standards:

ISO 9241 sub-metrics computed via BeautifulSoup4 (30% weight)
Nielsen 10 Heuristics computed via BeautifulSoup4 (30% weight)
WCAG 2.2 accessibility checked via Axe-core Node tool (40% weight)


Scores combine into a weighted composite score from 0-100
The system detects which standard scored lowest (weakest dimension)
A targeted refinement prompt is sent to the LLM to fix that specific issue
Steps 3-7 repeat until score reaches 85% or 5 iterations complete

Installation
Step 1 - Install Python packages
pip install -r requirements.txt
This installs:

beautifulsoup4 — parses HTML files so Python can search through them
lxml — faster HTML parser used by BeautifulSoup internally
requests — makes HTTP calls to the Ollama local API
jinja2 — renders the final usability report as HTML
rich — prints coloured score tables in the terminal


Step 2 - Install Node.js (needed for Axe-core WCAG checking)
Download and install Node.js LTS from: https://nodejs.org
Then install the Axe-core CLI tool globally:
npm install -g @axe-core/cli
Test it works:
axe --version
Axe-core is an industry-standard accessibility testing library.
It checks the generated HTML against WCAG 2.2 rules and returns
a list of violations with impact levels (critical, serious, moderate, minor).
The wcag_metrics.py file calls this as a subprocess and converts
those violations into a penalty-weighted score.

Step 3 - Install Ollama (the local LLM)
Download and install from: https://ollama.com
Then pull the Llama3 model:
ollama pull llama3
Test it works:
ollama run llama3 "say hello"
Ollama runs a local REST API at http://localhost:11434
The ui_generator.py file sends POST requests to this endpoint
with the UI generation prompt and receives HTML back.

Running the project
Once everything is installed:
python main.py
This will:

Load requirements from samples/sample_requirements.json
Generate a UI prototype using Ollama
Score it using the evaluation engine
Print the score breakdown in the terminal
Save the full report to outputs/score_report.json
